
EXTENSOES:


1) COMO INSTALAR AS EXTENSÕES:
    Clicar no menu do lado ESQUERDO da tela que possui três quadradinhos + um separado (extensions).
    Colar/digitar o nome da extensão EXATAMENTE como está aparecendo abaixo:


      EXTENSÕES:


- PHP Intelephense (ícone de um elefante com o nome do PHP): dá dicas para completar o código;

- Markdown All in One (ícone de um “M” com uma setinha apontando para baixo): Faz o preview de como o markdown está ficando;

- Bracket Pair Colorizer 2 (ícone parecido com dois arco-íris unidos): insere cor na abertura e fechamento das aspas, 
  além de inserir uma linha lateral entre as chaves;

- indent-rainbow: Insere um sombreamento em cada uma das indentações do código;

- Pomodoro Timer (ícone de um relógio azul partido ao meio): - Relógio que é exibido na parte inferior da tela
  para controlar o tempo que você está trabalhando (evitar fadiga);

- GitLens (serve para verificar quem fez modificações no seu código e há quanto tempo durante a sincronização do GitHub);

- vscode-faker (ícone de um homem com uma cartola preta): gera dados para testar o seu código (exemplo: nome, endereço,
  telefone, etc);

- ESLint: traz inteligência para o VSCode durante a programação em javacript. Deixa sua codificação mais rápida;

- Presentation Mode (ícone de uma tela de projeção):  exibe somente seu código em modo de apresentação – exibe demais
  informações da tela. Para ativá-la, pressione CTRL +SHIFT+ P e digite na barra o nome “Presentation Mode” (sem aspas).
  Para desativá-la, pressione a tela ESC;

- vscode-icons (ícone com o fundo cinza escuro e um detalhe em azul): Mostra os ícones dos arquivos de acordo com cada tipo.
  Por exemplo, arquivos com a extensão .php são exibidos com o símbolo da linguagem PHP (um círculo na cor roxa e as letras
  “PHP”) e assim por diante;

 - Todo Tree (ícone de uma arvore): faz marcação diferenciada no código ao inserir as palavras-chave TODO (algo a fazer 
   depois) e FIXME (algo a corrigir no código – usado para problemas em seu código);

 - Auto Close Tag: Ao abrir um tag, ele automaticamente insere a tag de fechamento – traz produtividade;

- Auto Rename Tag: Caso você tenha digitado alguma tag errada, basta você corrigir a tag de abertura, pois este plugin 
  renomeia automaticamente a tag de fechamento do HTML. É necessário adicionar um parâmetro no arquivo settings.json;

- Format HTML in PHP – Formata o Código HTML dentro do PHP;

- PHP Getters & Setters – Automatiza a chamada dos métodos get e set das classes do PHP de forma mais fácil;

- Theme Monokai – Já vem instalado por padrão – Não é necessário instalar

- PHP debug (versão 1.13.0) – Permite fazer o debug do código PHP dentro do VSCode. Necessário adicionar algumas linhas no
  fim do arquivo php.ini (ver no vídeo “Instalação e Configuração do VSCode no Linux - Parte 3” – por volta de 05:00min).

- PHP Code Sniffer phpcs: Faz uma análise, estatística de qualidade do código;

- PHP DocBlocker: Permite fazer a documentação automática do código;

- PHP Snippets: Ao digitar um comando (atalho), ele constrói uma estrutura completa do PHP no código;

- Minify: Minificar os arquvos css e js dentro do seu código;

- Wakatime: Mostra o tempo que você está programando – necessário criar uma conta no site do Wakatime
  (http://www.wakatime.com). SUGESTÃO: Utilizar a conta do GitHub no Wakatime. Assistir o vídeo “Instalação e 
  Configuração do VSCode no Linux - Parte 4” por volta dos 4:30s para maiores detalhes.

- Firacode: São fontes (letras) adicionais para o VSCode. O professor instalou e verificou que não ficou
  muito bom o resultado.

- Prettier (ícone de uma letra “P” colorida): Formatar seu código de forma mais organizada.
  Sugestão do professor: instalar o plugin Node.JS. Necessário configurações no arquivo settings.json (ver final do vídeo 5 – início do vídeo 6)

- Settings Sync: Sincroniza nossas configurações do VSCode no GitHub
